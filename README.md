# 📌 Projects

## 1. 비행기 별점 데이터를 활용한 문제점 분석 및 개선 방향 도출

### Overview
비행기 이용자의 별점 데이터를 분석하여 서비스의 문제점을 파악하고,  
개선 방향을 도출하기 위해 진행한 데이터 분석 미니프로젝트입니다.

단순히 별점 데이터를 확인하는 것에 그치지 않고,  
고객 만족도에 영향을 주는 요소를 분석하여 실제 개선 방향으로 연결하는 것을 목표로 했습니다.

### Role
코드 통합 및 최종 실행 검수 담당

### What I Did
- 팀원들이 작성한 데이터 처리, 분석, 시각화 관련 코드를 취합
- 전체 코드 실행 흐름 정리
- 데이터 불러오기, 전처리, 분석, 결과 확인 과정 점검
- 실행 중 발생하는 오류, 변수명 불일치, 코드 순서 문제 수정
- 분석 결과가 문제점과 개선 방향으로 연결될 수 있도록 전체 흐름 점검

### Tech Stack
Python, pandas, NumPy, 데이터 전처리, 데이터 분석, 데이터 시각화, Jupyter Notebook

### What I Learned
이 프로젝트를 통해 데이터 분석은 단순히 결과값을 확인하는 것이 아니라,  
분석 결과를 실제 문제점과 개선 방향으로 연결하는 과정이 중요하다는 것을 배웠습니다.

또한 팀 프로젝트에서 여러 사람이 작성한 코드를 하나로 합칠 때,  
데이터 흐름과 실행 순서를 정확히 관리하는 것이 최종 결과물의 완성도에 큰 영향을 준다는 점을 경험했습니다.

## 2. PPT 기반 강의 설명 AI 강사 제작

### Overview
사용자가 PPT 파일을 입력하면 슬라이드 내용을 분석하고,  
이를 바탕으로 강의하듯 설명해주는 AI 강사를 만드는 프로젝트입니다.

PPT의 텍스트, 이미지, 표 등 다양한 정보를 활용하여 슬라이드별 설명문을 생성하고,  
자연스러운 강의 흐름을 구성하는 것을 목표로 했습니다.

### Role
워크플로우 코드 통합 및 최종 실행 검수 담당

### What I Did
- 팀원들이 작성한 PPT 분석, 설명문 생성, 검색, 음성/영상 생성 관련 코드 통합
- 각 단계의 결과가 다음 단계로 정상 전달되는지 확인
- 슬라이드 데이터, 상태값, 파일 경로, 출력 결과 관련 오류 수정
- 전체 워크플로우가 끊기지 않고 실행될 수 있도록 코드 구조 정리
- 최종 실행 가능한 형태의 코드 완성

### Tech Stack
Python, LLM, AI Agent, Workflow 설계, PPT 분석, Gradio, TTS, 영상 생성 자동화

### What I Learned
이 프로젝트를 통해 AI 서비스는 개별 기능 구현뿐만 아니라,  
입력부터 결과 생성까지 이어지는 전체 파이프라인 설계가 중요하다는 것을 배웠습니다.

특히 PPT 분석, 설명문 생성, 음성/영상 생성처럼 여러 단계가 연결된 구조에서는  
상태 관리와 파일 경로 관리가 전체 실행 안정성에 큰 영향을 준다는 점을 경험했습니다.

## 3. Supervisor Node 기반 화장품 리뷰 분석 및 치명적 리뷰 도출

### Overview
화장품 리뷰 데이터를 분석하여 제품의 치명적인 문제점을 찾고,  
이를 바탕으로 개선 방향을 도출하기 위해 진행한 AI Agent 기반 프로젝트입니다.

Supervisor Node를 활용하여 여러 분석 흐름을 관리하고,  
리뷰 속 부정적 요소와 핵심 문제를 파악하는 것을 목표로 했습니다.

### Role
그래프 관리자 / 코드 통합 및 최종 실행 검수 담당

### What I Did
- 팀원들이 작성한 리뷰 분석 Agent 코드들을 취합
- Supervisor Node를 중심으로 전체 그래프 구조에 맞게 코드 통합
- 각 노드의 실행 흐름이 자연스럽게 이어지는지 확인
- 각 Agent의 입력값, 출력값, 상태값 전달 과정 점검
- 변수명 불일치, 상태값 누락, 노드 연결 오류, 결과 저장 문제 수정
- 최종적으로 화장품 리뷰 분석 결과와 치명적 리뷰 도출 과정이 정상 실행되도록 코드 정리

### Tech Stack
Python, LLM, LangGraph, AI Agent, Multi-Agent, Supervisor Node, 리뷰 데이터 분석, Streamlit

### What I Learned
이 프로젝트를 통해 Multi-Agent 시스템에서는 개별 Agent의 성능뿐만 아니라,  
전체 그래프 흐름과 상태 관리가 매우 중요하다는 것을 배웠습니다.

특히 Supervisor Node를 활용하면 여러 Agent의 역할을 분리하고,  
분석 흐름을 체계적으로 관리할 수 있다는 점을 경험했습니다.

또한 리뷰 데이터를 단순히 긍정/부정으로 나누는 것이 아니라,  
치명적 문제를 찾아 실제 개선 방향으로 연결하는 과정이 중요하다는 것을 배웠습니다.
